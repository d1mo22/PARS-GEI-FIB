# ER - Enginyeria de Requisits

<details open>
<summary>Català</summary>

## Sobre l'Assignatura

Assignatura obligatòria de 6 crèdits per a l'especialitat de Sistemes d'Informació que se centra en determinar quins sistemes de software s'han de construir. Com s'indica, "la tasca més difícil en construir un sistema d'informació és determinar quin sistema construir."

### Temes Principals
- Fonaments i activitats d'enginyeria de requisits
- Anàlisi de stakeholders i definició d'objectius
- Context del sistema, límits i modelització de casos d'ús
- Mètodes de determinació de requisits i resolució de conflictes
- Modelització conceptual (esquemes estructurals i de comportament)
- Traçabilitat i consistència entre artefactes
- Pràctiques de validació i documentació

### Objectius d'Aprenentatge
- Definir objectius del projecte i identificar stakeholders
- Crear casos d'ús i especificar requisits funcionals/no funcionals
- Realitzar argumentació de satisfacció vinculant requisits amb objectius
- Desenvolupar esquemes conceptuals a partir de requisits
- Validar especificacions mitjançant mètodes d'inspecció

## About the Subject

This is a 6-credit mandatory course for the Information Systems specialization that focuses on determining what software systems should be built. As noted, "the most difficult task in constructing an information system is determining which system to construct."

### Main Topics
- Requirements engineering fundamentals and activities
- Stakeholder analysis and objective definition
- System context, boundaries, and use case modeling
- Requirements determination methods and conflict resolution
- Conceptual modeling (structural and behavioral schemas)
- Traceability and consistency across artifacts
- Validation and documentation practices

### Learning Objectives
- Define project objectives and identify stakeholders
- Create use cases and specify functional/non-functional requirements
- Perform satisfaction argumentation linking requirements to goals
- Develop conceptual schemas from requirements
- Validate specifications through inspection methods
